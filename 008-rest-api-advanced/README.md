# 008-rest-api-advanced

REST APIの設計・運用観点を実務レベルに引き上げるステップです。

## ゴール
- レイヤード構成（handler/usecase/repository）を説明できる
- 認証・認可・バリデーション・エラーハンドリングを統一設計できる
- OpenAPIベースでAPI仕様を管理できる

## 学習トピック
- ルーティングとmiddleware設計
- DTOとドメインモデルの分離
- 認証（JWT/OAuthの概念）
- バリデーションと一貫したエラーレスポンス
- OpenAPIとドキュメント運用

## 実装課題（自分で実装）
1. TODO APIをクリーンな層分離で再実装
2. 認証付きエンドポイントとロール制御
3. OpenAPI仕様と実装差分チェック手順を作る

## 自分への確認質問
- handlerにビジネスロジックを書くと何が困る？
- 認証と認可の違いは？
- API互換性を壊さない変更戦略は？
