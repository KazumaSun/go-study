# 003-errors-and-testing

実務で差がつく「エラー設計」と「テスト基礎」を固めるステップです。

## ゴール
- ラップされたエラーを追跡できる
- `table-driven test` を自然に書ける
- モック過多にならないテスト戦略を説明できる

## 学習トピック
- `errors.New` / `fmt.Errorf` / `%w`
- `errors.Is` / `errors.As`
- カスタムエラー型
- `testing`パッケージ、サブテスト、`t.Helper`
- ベンチマーク（`go test -bench`）の基礎

## 実装課題（自分で実装）
1. バリデーション層でエラー分類を行う
2. ユースケース層にtable-driven testを作る
3. 失敗ケース中心のテスト（境界値）を追加する

## 自分への確認質問
- `%w` と `%v` の違いは？
- `errors.Is` が真になる条件は？
- テストで「何をモックしないか」をどう決める？

## 完了条件
- 正常/異常の主要分岐にテストがあり、エラー伝播が説明できる
