# 013-rest-api-production

REST APIを実務レベルで運用できる状態にするカリキュラムです。

## ゴール
- REST APIの設計原則（リソース設計、冪等性、ステータスコード）を説明できる
- ページング、フィルタ、ソート、バージョニングを実装できる
- レート制限、認証、監査ログを組み込んだAPIを設計できる

## 学習トピック
- URI設計、HTTPメソッドの責務
- エラーレスポンス標準化
- offset/limit と cursor pagination
- OpenAPI運用と後方互換
- 負荷試験とSLOの設定

## 実装課題
1. 商品API（一覧/詳細/作成/更新/削除）を実装する
2. 一覧APIにページング・フィルタ・ソートを導入する
3. APIバージョニング（`/v1`）と後方互換方針を定義する

## 完了条件
- API設計レビューで「なぜそのエンドポイント設計か」を説明できる
